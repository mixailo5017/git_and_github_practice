Navigation Bar

The Navigation Bar was designed to allow the modular addition of navigation assets aligned to both sides of a fluid-width layout. Navigation options include dropdown menus and forms.

Standard Navbar

*note: This display model of the main page navbar was due to the fact that the navbar itself while being fluid can not accomodate an indefinite amount of content. Because this model is contained within a fixed width example element, the number of menu items displayed had to be condensed.

Navigation Bar Components

Alignment:

The navigation bar can display content aligned either to the right or left hand side. By default, everything is left hand aligned. To align content to the right, add the 'm-right' class to the navigation tag.

In the event that you require multiple separate right-hand aligned menus, bear in mind that they align right to left in decending order in the layout. It's counter-intuitive, but keep in mind that the first m-right asset will be the one furthest to the right.

Two menus have a dedicated purpose as the responsive menus in the main header. These styles should not be repeated outside of the main header.

Main Menu

*note: In order to get the desired layout on the site, the menu position had to be set to 'fixed', and is in not visible in this example. I apologize for the inconvenience.

The main menu is toggled by a button that only appears in mobile viewports. The button currently appears to the left of the logo. Scale the browser down to see an example of the button.

Separators

Line spacing is accomplished by applying the 'separator' class to a list item. This will appear as a vertical line on the standard menus and a horizontal line on dropdowns and mobile menus.

On the Main Menu, line breaks are automatically appended below menu items, and separators are hidden from view.
